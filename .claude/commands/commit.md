# Commit

ステージされた変更を確認し、プロジェクトのコミットメッセージ規則に従ってコミットします。

## タスク

### 1. 変更内容の確認

以下を実行して変更内容を把握する:

```bash
git diff --staged
git status
```

ステージされた変更がない場合は、その旨を伝えて終了する。

### 2. コミットメッセージの提案

以下の規則に従ってコミットメッセージを生成する:

**フォーマット**: `<type>[<scope>]: <説明>`

**type**:
- `feat` - 新機能・機能追加
- `fix` - バグ修正
- `refactor` - 動作を変えないコードの整理
- `test` - テストの追加・修正
- `docs` - ドキュメントのみの変更
- `calc` - 実験・検証用コードの追加

**scope**（変更対象のコンポーネント）:
- `opt` - layout-design-opt
- `backend` - layout-design-backend
- `frontend` - layout-design-frontend
- 複数にまたがる場合はscopeを省略してよい

**説明**: 日本語で一行、簡潔に記述する

### 3. 確認と実行

提案したコミットメッセージをユーザーに提示し、承認を求める。
承認されたら以下を実行する:

```bash
git commit -m "<承認されたメッセージ>"
```

ユーザーがメッセージを修正した場合は、修正後のメッセージでコミットする。
